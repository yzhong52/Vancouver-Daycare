# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Set up environment
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# Run the full data refresh pipeline
.venv/bin/python refresh.py

# Run a single pipeline step
.venv/bin/python download_childcare_pdfs.py
.venv/bin/python parse_pdfs.py
.venv/bin/python enrich_vacancies.py
.venv/bin/python geocode_vacancies.py

# Serve the site locally
python3 -m http.server    # then open http://localhost:8000/index.html
```

No test suite, no linter, no build step.

## Architecture

This is a **static GitHub Pages site** (`vandaycare.ca`) with a Python data pipeline that feeds it. There is no backend server at runtime.

### Data pipeline (Python)

Four scripts run in sequence via `refresh.py`. GitHub Actions triggers this every 48 hours and opens a PR when `processed_data/` changes.

```
download_childcare_pdfs.py  →  data/*.pdf  (gitignored)
parse_pdfs.py               →  processed_data/providers.csv + vacancies.csv
enrich_vacancies.py         →  processed_data/enriched_vacancies.csv
geocode_vacancies.py        →  processed_data/geocoded_vacancies.csv
```

- **Source data:** PDFs from [wstcoast.org](https://wstcoast.org) (WCCRC weekly vacancy list + all-providers directory)
- **Provider matching** in `enrich_vacancies.py`: phone → email → fuzzy name (rapidfuzz, 75% threshold). Unmatched entries are RLNR home-based providers not in the directory.
- **Geocoding**: BC Address Geocoder (`geocoder.api.gov.bc.ca`) — free, no API key. Appends ", Vancouver, BC" to each address string.
- `processed_data/` CSVs are committed to git so week-over-week diffs are visible in PRs.

### Frontend (vanilla JS + Leaflet)

`index.html` is the entire frontend — no framework, no build tool. It:
1. Loads **two data sources in parallel**:
   - `processed_data/geocoded_vacancies.csv` — auto-generated by the pipeline (blue pins)
   - `vacancies.json` — manually curated entries (orange pins)
2. Merges them into `allData` and renders Leaflet markers with custom SVG `divIcon`s
3. Filters by child's birthday using age range strings in formats `"2.5 - 5 yrs"` and `"19 - 36 mo"`

The `_source` field (`"auto"` vs `"manual"`) controls pin color. Manual entries without lat/lng are silently skipped.

### Manual vacancies system

`processed_data/additional_vacancies.json` (committed to git) holds manually curated vacancy entries as a JSON array. `admin.html` is a self-contained admin UI to manage this file:

- Loads `vacancies.json` via `fetch` on page load
- Add/Edit/Delete entries via a modal form
- **Geocode button** calls the BC geocoder API directly from the browser
- **Save to GitHub**: uses the GitHub Contents API (PUT) with a PAT stored in `localStorage` — avoids needing the CLI to commit changes
- **Download JSON**: fallback for committing manually

The `vacancies.json` schema mirrors the CSV column names used by the frontend (`name`, `address`, `neighbourhood`, `phone`, `email`, `website`, `languages`, `age_groups`, `curriculum`, `contact_date`, `vacancies`, `lat`, `lng`) plus `id`, `notes`.

### Deployment

- GitHub Pages serves the repo root; `CNAME` sets the custom domain `vandaycare.ca`
- No build step — `index.html` and `vacancies.json` are served directly
- GA4 tracking ID `G-GM16Z1VGT9` is hardcoded in `index.html`
